# Upgrade Policy

When and how technology choices are re-evaluated. Triggers (any one fires an upgrade review in `lifecycle/`):

- new major version
- security advisory
- end of official support / maintenance stall
- license change
- performance budget exceeded
- operational cost increase
- a clearly advantageous alternative emerges
- architectural assumptions behind the choice change
- (AI models) a new major provider release

Output of a review: an `upgrade-review` doc + an updated ADR (or a new one superseding the old).
