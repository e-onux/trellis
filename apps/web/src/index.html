The problem Trellis solves

Spec tools tell an agent what to build. Trellis preserves why it was built that way, how each piece is verified, how much it can grow, and when it must be reconsidered.

Unbounded growth

Duplicated responsibilities, runaway dependencies, ballooning files.

Architecture drift

Locally sensible changes that are globally inconsistent.

Lost rationale

Decisions live in chat history, not in the repository.

Hidden tests

Only engineers can see whether a feature actually works.
